# 梦核跑酷 — 专注力驱动跑酷游戏

`index.html` 是本作品的核心前端文件，无需构建工具，直接在浏览器中运行。

---

## 2  作品整体方案设计与实现

### 2.1  整体技术路线

```
EEG设备 / 陀螺仪
      ↓
Python 后端（数据采集 + WebSocket 服务）
      ↓  ws://localhost:8765  JSON帧
浏览器前端（Three.js 跑酷游戏）
      ↓
注意力 → 游戏速度 / 场景切换 / 积分
陀螺仪 → 角色横移 / 跳跃
```

系统分为两层：Python 后端负责采集 EEG 和姿态传感器数据并推送；前端单页面应用（纯 HTML + JS）接收数据，实时驱动 Three.js 3D 跑酷游戏。前端不依赖任何构建步骤，所有库通过 CDN 引入（Three.js r145、GSAP、EffectComposer、GLTFLoader 等）。

---

### 2.2  核心技术方案设计

#### 2.2.1  后端数据解析方案

后端以 `ws://localhost:8765` 建立 WebSocket 服务，每帧推送如下 JSON：

```json
{
  "gyroscope_x": 170.2,
  "gyroscope_y": 268.5,
  "gyroscope_z": 171.0,
  "attention": 62,
  "focus_x": 0.48,
  "focus_y": 0.51,
  "focus_area_width": 120,
  "focus_area_height": 90
}
```

前端收到后执行两步归一化：

- **注意力**：若 `attention > 1` 则直接使用（百分制），否则乘以 100。结果裁剪至 `[0, 100]`。
- **陀螺仪**：以 `GYRO_Z_REST = 170`（Z轴静止值）为基准判断左右移动，以 `GYRO_Y_REST = 270`（Y轴静止值）为基准判断跳跃。超出阈值时触发一次性事件（边缘触发，非电平触发），避免连续误判。

无硬件时，页面内置模拟面板，可用滑块手动设定注意力值，键盘 ←→ 控制横移、↑ 控制跳跃，保证开发与演示期间可独立运行。

#### 2.2.2  前端游戏逻辑方案

游戏核心基于 **Three.js r145**，采用单帧循环 `animate()` 驱动：

| 子系统 | 实现方式 |
|---|---|
| 无限地面 | `groundSegments[]` 环形复用，超出镜头后移至前方 |
| 障碍物 | 按注意力调节生成频率；切换场景后使用对应主题几何体 |
| 碰撞检测 | 每帧 AABB 包围盒检测（`Box3.intersectsBox`） |
| 速度系统 | `baseSpeed = 0.05 + 0.001 × att^1.2`，分三段加速；区间切换后延迟 800ms 应用新速度 |
| 金币积分 | 收集金币 → `coinCount` 累加 → `getFocusPct()` 换算为 0–100% 进度 |
| 后处理 | `EffectComposer` + `UnrealBloomPass` + `VignetteShader`，场景切换时动态调参 |
| 觉醒机制 | 注意力持续 >30 或金币达 100 时触发，短暂爆发 Bloom 与全屏闪光 |

GLB 模型（树木、建筑、道路）通过 `GLTFLoader` + `DRACOLoader` 异步加载，加载完成后缓存为模板并在运行时 `clone()` 复用，保持零重复解析开销。加载失败时自动回退至程序化几何体，不影响可玩性。

#### 2.2.3  梦核场景切换方案

游戏共设 **5 段场景**，由金币积分百分比驱动切换：

```
0 % ──→ 30 %   梦核主跑道（粉紫色调、程序化地面）
                      ↓ 30% 触发
              三扇发光大门弹出（左/中/右对应三条车道）
              玩家跑入对应门即进入子场景
  ┌─────────────────────────────────────────┐
  │ 海上城市  │  草地    │  星系            │
  │ 水面着色器│ eztree   │ 螺旋星系粒子     │
  │ GLB道路   │ 草地贴图 │ 霓虹GLB赛道      │
  │ 海景建筑  │ 树/岩石  │ 漂浮晶体         │
  └─────────────────────────────────────────┘
                      ↓ 70% 统一接回
              原神终章（Genshin 复合场景）
                      ↓ 100%
              游戏结束 + 专注力分析报告
```

**场景切换实现细节：**

1. **三扇门动画**（`triggerSceneGate`）：在三条车道前方生成发光门框（`LineSegments` + 点光源），5秒内未选择则强制进入原神。
2. **场景入驻**（`applyScene(key)`）：隐藏原有程序化地面，切换天空/雾/Bloom/暗角预设，按场景类型启动专属单例（水面、星系、草地）。
3. **原神复合渲染**（`enterGenshinScene`）：启动独立 `gScene` + `gCamera` + `gComposer`，每帧先渲染原神场景，清除深度缓冲后在上方叠加跑酷角色层，两层通过 `G_RUNNER_SCALE = 23/6` 的比例系数对齐坐标系。

**各子场景技术要点：**

- **海上城市**：自定义水面 `ShaderMaterial`（正弦波顶点位移 + 深/浅色混合），Kenney CC0 `road_square` 片段无缝平铺为跑道，Quaternius CC0 建筑沿两侧生成天际线。
- **草地**：eztree 草地贴图（`ground_grass.jpg`，1024×1024）铺 800×800 平面，`texture.offset` 每帧随相机位置更新实现世界锁定滚动；eztree 原仓库 `grass.glb`、`rock1-3.glb`、`flower_*.glb` 随机散布（Draco 解压支持）。
- **星系**：7000 粒子螺旋星系（`BufferGeometry` + `ShaderMaterial`）+ wave 网格折叠动画；道路复用 `road_square` 但材质替换为自发光紫/青，营造霓虹宇宙赛道。

### 2.3  核心创新点

1. **生理信号直接映射游戏物理**：注意力数值实时控制奔跑速度与横向灵敏度，专注程度越高角色移动越流畅，形成即时正反馈回路。

2. **多维感知融合控制**：EEG 专注力 + 陀螺仪姿态双通道并行输入，分别负责"意图强度"与"方向操控"，无需手柄或键盘即可完成完整的跑酷游玩。

3. **积分驱动的渐进式叙事**：场景不依赖时间切换，而由玩家专注状态（金币积分）驱动，专注越稳定越快解锁下一段视觉旅程，量化了心流体验。

4. **零构建单页部署**：所有 Three.js 后处理、GLTFLoader、DRACOLoader 均通过 CDN 引入，GLB 模型直接放置本地目录；作品在任意静态服务器下即可运行，降低了展示门槛。

5. **双场景复合渲染**：原神终章通过"清除深度缓冲再叠渲染"技术将跑酷角色精确融合进独立的 Three.js 场景，实现两套坐标系下的无缝视觉衔接。

## 3  作品测试方案与测试结果

### 3.1  整体测试环境

| 项目 | 配置 |
|---|---|
| 操作系统 | Windows 11 |
| 浏览器 | Chrome 124+（含 Headless Chrome + SwiftShader WebGL 用于自动化） |
| 静态服务 | Node.js `http.createServer` 本地文件服务（端口 8080） |
| WebSocket | Python `websockets` 库，端口 8765 |
| 硬件模拟 | 页面内置模拟滑块 + 键盘回退，无需物理设备即可全流程测试 |

---

### 3.2  详细测试方案

**① WebSocket 数据链路测试**

向 `ws://localhost:8765` 分别推送边界值（`attention=0`、`attention=1`（小数归一）、`attention=100`、陀螺仪越过跳跃/换道阈值），验证：
- 注意力归一化逻辑（>1 直接用，≤1 乘100）正确无误
- 速度随注意力连续变化，无阶跃跳变
- 陀螺仪边缘触发换道和跳跃仅触发一次

**② 场景切换触发测试**

通过模拟滑块将注意力维持高位，逐步积累金币至 30%、70%、100%，验证：
- 30%：三扇门准时弹出，每扇门与对应车道对齐
- 玩家跑入各门：`currentScene` 正确切换为 `'sea'`/`'grass'`/`'galaxy'`，场景专属视觉正常加载
- 70%：无论当前处于哪个子场景，`enterGenshinScene()` 被调用，原神复合场景启动
- 100%：游戏结束画面与专注力报告正常显示

**③ GLB 模型加载测试**

使用 Headless Chrome CDP 脚本在页面加载后注入检查代码，断言：
- `sceneRoadTiles.length > 0`（海/星系道路已生成瓦片）
- `grassBladeTemplates.length > 0` 且 `rockTemplates.length > 0`（草地 eztree 模型已解压）
- `sceneBuildingTemplates.length > 0`（建筑模板已就绪）
- 运行时零异常

测试结果：海上城市 8 条道路瓦片 + 水面 + 30 个建筑装饰；星系 7 条瓦片 + 粒子系统；草地 46 个 eztree 装饰物；全程零运行时异常。

**④ 碰撞与积分测试**

键盘模式下手动碰撞障碍物，验证：
- 碰撞后 `coinCount` 扣减，积分条下降
- `hitCooldown` 无敌帧期间不重复扣分
- 拾取金币时飞行粒子动画播放，`getFocusPct()` 数值同步更新

**⑤ 性能基线测试**

Chrome DevTools Performance 录制 30 秒游戏过程（草地场景，装饰物最多），观察：
- 帧率维持 55 FPS 以上（60 Hz 显示器）
- 无持续性内存增长（GLB 克隆对象正常释放）
- 场景切换时帧率抖动不超过 3 帧
